什么是单峰函数？

肯定要先听听度娘的解释啦！

单峰函数是在所考虑的区间中只有一个严格局部极大值(((峰值)))的实值函数。如果函数f(x)f(x)f(x)在区间[a,b][a, b][a,b]上只有唯一的最大值点CCC，而在最大值点CCC的左侧，函数单调增加；在点CCC的右侧，函数单调减少，则称这个函数为区间[a,b][a, b][a,b]上的单峰函数。——百度百科

就是函数的右边单调下降，左边单调上升，也可以这样解释：

如果f(x)f(x)f(x)最大：

  • 如果a<b<xa<b<xa<b<x，那么f(b)≥f(a)f(b)\ge f(a)f(b)≥f(a)。
  • 如果x>a>bx>a>bx>a>b，那么f(a)≥f(b)f(a)\ge f(b)f(a)≥f(b)。

暴力枚举

最普通的方法肯定是暴力啦！

这个不用说了吧，设置步长，从左到右扫记录最大值。

优点：相对稳定。

缺点：耗时长，精度低。

#include<bits/stdc++.h>
using namespace std;
const int pre=2;//精度可调
int main(){
    double p=pow(10,-pre);//计算步长
	double l=0,r=10,maxx=-1;//看情况来定maxx的初始值
	for(double i=l+pre;i<=r;i+=p)//由于使用x^(1/x)为f函数，所以i不能为0
    	maxx=max(maxx,f(i));//寻找最大值
    cout<<fixed<<setprecision(pre)<<maxx;
}

时间复杂度：O(10n)O(10^n)O(10n)，照着这样，111秒只能算到777位数左右。

三分查找111

像这种带有单调性的函数，让人不由自主的想起——二分。

但是我们不能只取一个点，因为这是无法得出峰值在哪里的，所以我们需要取两个点，通过比较这两个点来得出峰值的位置。

俩学家阮大佬曾在他的博客中介绍了三分查找，主要是分一半再分一半的思想（midmidmid是lll和rrr的中点，mmidmmidmmid是midmidmid和rrr的中点）。如图：

比较f(mid)f(mid)f(mid)和f(mmid)f(mmid)f(mmid)的值：

  • 当f(mid)>f(mmid)f(mid)>f(mmid)f(mid)>f(mmid)的值时，使r=mmidr=mmidr=mmid。即峰值必在middmiddmidd左边。
  • 当f(mid)<f(mmid)f(mid)<f(mmid)f(mid)<f(mmid)时，使l=midl=midl=mid。即峰值必在midmidmid右边。

证明在链接里有，这里就照着我的思路再讲一遍吧（其实是差不多的）：

当f(mid)>f(mmid)f(mid)>f(mmid)f(mid)>f(mmid)的值时，若峰值在middmiddmidd右边，会导致有222个峰（否则f(mid)f(mid)f(mid)不会大于f(mmid)f(mmid)f(mmid)），所以峰值必在middmiddmidd左边。

同样可以证第二个。

既然这样，我们就可以进行三分，每次可以舍去原长度的12\frac{1}{2}21​或14\frac{1}{4}41​。

那么时间复杂度最优O(2log⁡2n)O(2\log_2n)O(2log2​n)，最坏O(2log⁡43n)O(2\log_\frac{4}{3}n)O(2log34​​n)。

另外，在链接里有提及到写法，一种是控制迭代次数，一种是直接控制精度，接下来的本节的代码都是直接控制精度。

优点：速度快。

#include<bits/stdc++.h>
using namespace std;
const int pre=10;//设置精度
int main(){
	double p=pow(10,-pre);//计算精度限制
    double l=0,r=10,mid,mmid;//初始化
    while(r-l>p){
        mid=(l+r)/2;//计算mid
        mmid=(mid+r)/2;//计算mmid
		if(f(mid)-f(mmid)>=0)r=mmid;//大于时舍去右边
		if(f(mid)-f(mmid)<=0)l=mid;//小于时舍去左边
    }
    cout<<fixed<<setprecision(pre)<<max(f(l),f(r));//输出更大的
}

这里有一个问题：

代码中101010到111111行中，照着思路来说，应该是

else l=mid;

为什么要换成

if(f(mid)-f(mmid)>=0)r=mmid;
if(f(mid)-f(mmid)<=0)l=mid;

因为我在测试的过程中，发现若换成带else的代码，那么当pre>15pre>15pre>15的时候，由于C++C++C++的精度问题会将x+r2\frac{x+r}{2}2x+r​的值算成lll（xxx很接近lll），那么就会算出mid=lmid=lmid=l，然后又算出mmid=lmmid=lmmid=l。那么mid=mmidmid=mmidmid=mmid就算出了f(mid)−f(mmid)f(mid)-f(mmid)f(mid)−f(mmid)等于000，那么就会执行else后的语句l=midl=midl=mid，然而midmidmid本来就等于lll，也就是说什么都没有变，最终导致了死循环。

另外，我们并没有提到过当f(mid)=f(midd)f(mid)=f(midd)f(mid)=f(midd)时的情况，因为在此时峰值三种情况都有可能（如图），所以只能靠“蒙”。

三分查找222

为了让算法变得更稳定，我们可以把这两个点设在三等分处，这样每次都一定会舍弃原长度的13\frac{1}{3}31​。时间复杂度O(2log3n)O(2log_3n)O(2log3​n)。

优点：速度快，相对更稳定。

#include<bits/stdc++.h>
using namespace std;
const int pre=10;
int main(){
	double p=pow(10,-pre);
    double l=0,r=10,m1,m2;
    while(r-l>p){
        m1=(l+r+l)/3;
        m2=(l+r+r)/3;//这两句需要注意！！！
		if(f(m1)-f(m2)>=0)r=m2;
		if(f(m1)-f(m2)<=0)l=m1;
    }
    cout<<fixed<<setprecision(pre)<<max(f(l),f(r));
}

在代码中有一处是需要注意的。

千万不要写成m1=(l+r)3和m2=(l+r)2/3！！

很多人认为二分时m=12(l+r)m=\frac{1}{2}(l+r)m=21​(l+r)，然而自然就想到三分是应该是13(l+r)\frac{1}{3}(l+r)31​(l+r)和23(l+r)\frac{2}{3}(l+r)32​(l+r)

这样写是错误的，例如l=3,r=4l=3,r=4l=3,r=4的时候m1=73=2.333…,m2=143=4.666…m1=\frac{7}{3}=2.333\dots,m2=\frac{14}{3}=4.666\dotsm1=37​=2.333…,m2=314​=4.666…。根本不在lll和rrr的范围内！

因为我们截取的是一段，所以原长度的13\frac{1}{3}31​应该是r−l3\frac{r-l}{3}3r−l​，也就是说m1m1m1应该是l+r−l3=2l+r3l+\frac{r-l}{3}=\frac{2l+r}{3}l+3r−l​=32l+r​，m2=r−r−l3=2r+l3m2=r-\frac{r-l}{3}=\frac{2r+l}{3}m2=r−3r−l​=32r+l​。

那为什么二分是m=l+r2m=\frac{l+r}{2}m=2l+r​？

其实也是一样的道理，m=l+r−l2=l+r2m=l+\frac{r-l}{2}=\frac{l+r}{2}m=l+2r−l​=2l+r​。

“二分查找”

我们发现只要m1m1m1到lll的距离和m2m2m2的到rrr距离相等，算法就会稳定。如果我们想需要速度更快，应该怎么处理呢？

可以想到我们如果把m1m1m1和m2m2m2取得接近于中点，那么一次比较就可以舍去几乎12\frac{1}{2}21​的长度了！

如图中（CCC是一个较小的常数）：

优点：速度比普通三分快

缺点：语言浮点运算误差必须小

#include<bits/stdc++.h>
using namespace std;
const int pre=10;
int main(){
	double p=pow(10,-pre);
	double l=0,r=10,m;
	while(r-l>=p){
		m=(l+r)/2;//取中点 
		if(f(m-p)-f(m+p)>=0)r=m;
		if(f(m-p)-f(m+p)<=0)l=m;//比较中点两端 
	}
    cout<<fixed<<setprecision(pre)<<max(f(l),f(r));
}

注意：如果常数CCC太小，那么C++C++C++就会将mid−Cmid-Cmid−C和mid+Cmid+Cmid+C看作相等，然后l=mid,r=midl=mid,r=midl=mid,r=mid，然后结束循环，最终你得到的答案只是f(mid)f(mid)f(mid)的值。

时间复杂度大约是O(log⁡2n)O(\log_2n)O(log2​n)。

0.6180.6180.618法

然而这些做法都不能很好地优化。（除了求导二分）

最好的做法是循环利用上一次比较时剩下的点。使一个点被去掉后另一个点正好是下一次比较的其中一个点。这就是0.618法，是优选法的其中一种。

那么这两个点应该取在哪里呢？

AAA点在BBB点去掉后的那一段上的位置与BBB点原先在整段上所处的位置比例相同,那么这个比例通过计算得出：

ab\frac{a}{b}ba​约等于0.6180.6180.618。

那么a=r−l1+1+52a=\frac{r-l}{1+\frac{1+\sqrt{5}}{2}}a=1+21+5​​r−l​，也就是是r−lϕ+1\frac{r-l}{\phi+1}ϕ+1r−l​。

这样，我们就确定了两个点的位置了。

优点：重复利用测试点，速度极快。

#include<bits/stdc++.h>
#define F 0.38196601125010515179541316563436//1-φ
using namespace std;
const int pre=10;
int main(){
	double p=pow(10,-pre);
	double l=0,r=10,m1=l+F*(r-l),m2=r-F*(r-l);
	while(r-l>p){
		if(f(m1)>=f(m2)){
			r=m2;//去掉右边
			m2=m1;//替换
			m1=l+F*(r-l);//计算新的m1
			if(f(m1)>f(m2))continue;//如果等于先不要结束
		}
		if(f(m1)<=f(m2)){
			l=m1;//去掉左边
			m1=m2;//替换
			m2=r-F*(r-l);//计算新的m2
		}
	}
    cout<<fixed<<setprecision(pre)<<f(l);
}

int f(int x){

using namespace std;
double S,V,H;//表面积，体积，高
double f(double R){
	H=sqrt((S/(M_PI*R)-R)*(S/(M_PI*R)-R)-R*R);//通过表面积和底面半径算出高
	V=M_PI*R*R*H/3;//先存到V里
	return V;//返回体积
}
int main(){
	while(cin>>S){//有多个数据
		double l=0,r=sqrt(S/M_PI),m1,m2,t=50;//初始化
		/*因为表面积是πr(r+l)那么r大概就是sqrt(表面积/π)*/
		while(t--){//控制迭代次数，50次
			m1=(l+l+r)/3;
			m2=(l+r+r)/3;
			if(f(m1)<f(m2))l=m1;
			else r=m2;
			/*这部分基本一样*/
		}
		cout<<fixed<<setprecision(2)<<V<<endl<<H<<endl<<l<<endl;//照题意输出
	}
	return 0;
}

当然，你也可以三分hhh的值，只是通过SSS和hhh的值比较难求出rrr的值。

本章小结

其实一般的三分题，只需要：

  1. 记住模板
  2. 选择要三分的数
  3. 修改f(x)函数
  4. 设置好左端点和右端点
  5. 调整精度
